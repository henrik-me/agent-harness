## Goal

Fix the two coupled bugs identified in #290 — (a) the orphaned `sub-invaders-bootstrap-summary.md` and its four stale link sets, and (b) the cross-repo handoff doctrine that routed an update for a phantom consumer file — and close the underlying process gaps so neither class of failure recurs. Specifically: restore the lost file at a stable in-repo URL; declare it harness-internal-only in doctrine; teach the close-out procedure to preserve **all** files in a directory-form CS; and add a pre-flight check to cross-repo doctrine so the orchestrator never again files an issue to update a file that does not exist in the target repo.
